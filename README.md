# SP MedPortal

Внутренний портал сотрудников Сергиево-Посадской больницы.

## Текущее состояние

- `index.html` — стартовая страница
- `login.html` — экран входа
- `register.html` — заявка на регистрацию
- `forgot.html` — запрос на восстановление пароля
- `dashboard.html` — кабинет сотрудника
- `admin.html` — админ-панель
- `quality.html` — раздел `ОКК и БМД`
- `training.html` — раздел обучения
- `assets/app.js` — клиентская логика локального прототипа и защитные ограничения
- `supabase/schema_v1.sql` — схема таблиц и RLS-политик для безопасной серверной версии

## Важно про безопасность

Сайт сейчас намеренно переведен в режим `fail closed`:

- на локальном `file://` или `localhost` можно смотреть и тестировать прототип
- на опубликованном статическом сайте вход, регистрация, админка и восстановление пароля должны оставаться закрыты, пока не подключена серверная авторизация

Это сделано специально, чтобы не публиковать уязвимую псевдо-авторизацию на клиенте.

## Что уже усилено

- убраны прямые обходные ссылки в админку и демо-кабинет
- добавлены CSP-ограничения, `referrer policy` и `noindex`
- включена строгая проверка пароля: минимум `8` символов, буква и цифра
- добавлен авто-выход по бездействию
- опасный рендер пользовательских данных экранируется
- роли и доступы подготовлены под `Supabase RLS`

## Что нужно для боевого запуска

Перед публикацией как рабочего портала нужно доделать серверную часть:

1. Применить `supabase/schema_v1.sql`
2. Развернуть Edge Functions:
   - `submit-registration`
   - `admin-registration`
   - `admin-profile`
   - `admin-reset-password`
3. Назначить первого администратора в таблице `profiles`
4. Только после этого открывать вход на опубликованном домене

Для первого администратора есть шаблон:

- `supabase/bootstrap_admin.sql`

## Supabase

SQL-схема лежит в:

- `supabase/schema_v1.sql`

Конфиг функций:

- `supabase/config.toml`

Код функций:

- `supabase/functions/submit-registration/index.ts`
- `supabase/functions/admin-registration/index.ts`
- `supabase/functions/admin-profile/index.ts`
- `supabase/functions/admin-reset-password/index.ts`

Она содержит:

- таблицы пользователей, заявок, уведомлений, сообщений и сброса пароля
- роли `employee`, `okk_member`, `okk_head`, `admin`
- `Row Level Security` политики для ограничения доступа

## Как теперь устроен вход

- сотрудник регистрируется через серверную функцию
- система создает скрытую auth-учетную запись и держит ее заблокированной до одобрения
- администратор одобряет заявку
- после одобрения сотрудник может войти по своему логину и паролю
- админ может менять роль, статус и выдавать временный пароль через серверные функции
